# Deploying

Deployment follows a two-step promotion process: **staging → production**.

## Standard Deployment

1. Open a pull request to merge your changes into the `staging` branch.
2. When you're ready to release, open a pull request to merge `staging` into `production`.
   If more changes land in `staging` before that PR is merged, re-review the promotion PR because its release scope has changed.

## Manual Deployment

You can also [trigger the deployment workflow manually](https://github.com/Frigear-nu/nuxt-frigear-nu/actions/workflows/deploy.yml) from the Actions tab.

Click **Run workflow** and fill in the required fields.
