# Forms

You can define forms inside [`content/forms/**.yml`](../content/forms),
follow the schema from [`shared/schema/content-form.ts`](../shared/schema/content-form.ts).

> NOTE: For now you can only define forms with a hardcoded zod steppedForm.

## Examples

- [`shared/schema/forms/applications.ts`](../shared/schema/forms/applications.ts)

## Features

- [X] Define form schemas using Zod
- [X] Configure form using `content/forms/**.yml`
- [X] Delivery of forms
    - [X] Webhooks
    - [X] Email
- [ ] Define forms using yaml file notation

## Delivery

A single form can deliver to multiple destinations, email or webhook is supported for now.

### Schema

```yaml
delivery:
  - channel: email
    destination: [ "email@domain.lol", "another@domain.lol" ]
  - channel: webhook
    destination: [ "https://webhook.site/12345678" ]
    headers:
      Authorization: "Bearer ${secrets.WEBHOOK_SECRET}"
```

### Variables / Secrets

> This allows not setting this ins the yaml, since it is considered public.

Currently the only available variables are:

| `${variable}`             | Value                                | Description                |
|---------------------------|--------------------------------------|----------------------------|
| `${contact.default}`      | `runtimeConfig.mail.to`              | Default to email (contact) |
| `${contact.volunteering}` | `runtimeConfig.contact.volunteering` |                            |
| `${contact.partnership}`  | `runtimeConfig.contact.partnership`  |                            |
| `${contact.support}`      | `runtimeConfig.contact.support`      |                            |
| `${contact.payment}`      | `runtimeConfig.contact.payment`      |                            |
| `${contact.governance}`   | `runtimeConfig.contact.governance`   |                            |
| `${contact.finance}`      | `runtimeConfig.contact.finance`      |                            |
| `${contact.complaint}`    | `runtimeConfig.contact.complaint`    |                            |
| `${contact.other}`        | `runtimeConfig.contact.other`        |                            |

> To add more items, check out [`server/routes/api/forms/[...formPath].post.ts`](../server/routes/api/forms/[...formPath].post.ts)

#### Example

```yaml
delivery:
  - channel: email
    destination: [ "${contact.default}" ] 
```
